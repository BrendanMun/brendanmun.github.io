About the ternary ?: operator

The ternary operator provides a concise alternative to writing multi-line if...else statements in JavaScript. The syntax is as follows:

condition ? ifTrue : ifFalse

So instead of writing: 

const age = 21;
if (age >= 18) {
   console.log("You may purchase beer.")
} else {
   console.log("No beer for you.")
}
// Outputs: "You may purchase beer."

You could simply write:

const age = 21;
age >= 18 ? console.log("You may purchase beer.") : console.log("No beer for you.");
// Outputs: "You may purchase beer."

You could further shorten this code by declaring the tested variable on the same line as the ternary operator:

const age = 21 >= 18 ? console.log("You may purchase beer.") : console.log("No beer for you.");
// Outputs: "You may purchase beer."

Testing Booleans with the ternary operator

The ternary operator is typically used to assign one of two possible values to a single variable depending on some condition.

Below one of two possible greeting messages is displayed based on whether a user is logged in or not:

const isLoggedIn = true;
const greeting = isLoggedIn ? "Welcome, User!" : "Please log in to continue.";
console.log(greeting);
// Outputs "Welcome, User!"

And is this example the ternary operator provides a discounted price to members:

const isMember = true;
const price = isMember ? €9.99 : €12.99;
console.log(price);
// Outputs: €9.99

The test condition can include JavaScript's logical AND &&operator which requires that both conditions are true.

const isUserLoggedIn = true;
const isAdmin = false;

const accessLevel = isUserLoggedIn && isAdmin ? "Admin" : "Regular User";
console.log("Access Level: " accessLevel);
// Outputs: "Access Level: Regular user"

Testing strings with the ternary operator

See the following example.

// Variable declared on separate line
const weather = "rainy";
weather === "sunny" ? console.log("Go out.") : console.log("Stay home.");
// Outputs: "Stay home."

Or more concisely:

// Variable declared and tested on same line
const weather = "rainy" === "sunny" ? console.log("Go out.") : console.log("Stay home.");
// Outputs: "Stay home."

Testing numbers with the ternary operator

Here is an example of the ternary operator determining the larger of two numbers:

const num1 = 10;
const num2 = 20;
const largerNumber = num1 > num2 ? num1 : num2;
console.log("The larger number is:" +largerNumber);
//Outputs 20

Chaining and the ternary operator

In the example below a second ternary expression is chained to the false branch of the first ternary expression:

const myNumber = 10;
let testNum = myNumber === 0 ? "You have nothing" : myNumber > 10 ? "You have a lot" : "You have a little"; 
console.log(testNum);
// Outputs: "You have a little"

To make this code easier to read, you could rewrite it over multiple lines as follows:

const myNumber = 10;
let testNum = myNumber === 0 
    ? "You have nothing" 
    : myNumber > 10 
        ? "You have a lot"
        : "You have a little"
        
console.log(testNum);
// Outputs: "You have a little"

And here is a second example of chaining ternary operators.

const firstCheck = false;
const secondCheck = false;   

const grantAccess = firstCheck 
    ? "Access denied" 
    : secondCheck
        ? "Access denied" 
        : "Access granted";

console.log(grantAccess);
// Outputs: "Access denied"

Testing for 'problem' values with the ternary operator

You can use the ternary operator to check for values that may generate errors in your code. Typically, these are undefined or NULL values, or empty strings.

The code below will evaluate to true because the tested variable contains a value:

let isThereSoup = "Tomato soup";
let response = isThereSoup ? "Yes, we have soup": "No soup today."
console.log(response);
// Outputs: "Yes, we have soup"

However, if no value is assigned to the tested variable, the ternary expression will evaluate to false:

let isThereSoup;
let response = isThereSoup ? "Yes, we have soup": "No soup today."
console.log(response);
// Outputs: "No soup today."

In such cases you may want to assign some default value to the undeclared variable. See below:

let isThereSoup;
let response = isThereSoup ? "Yes, we have soup": "Default soup."
console.log(response);
// Outputs: "Default soup."

Below is an example of handling a NULL value:

const userInput = null;
const result = userInput !== null ? userInput : "Some default value";
console.log(result); // Output: "Some default value"

And here is an example of handling an empty string.

let userEmail = '';
let response = userEmail ? userEmail: "No email supplied."
console.log(response);
// Outputs: "No email supplied."
