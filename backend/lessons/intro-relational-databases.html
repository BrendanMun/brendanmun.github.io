About relationship databases

Here are some terms you will meet when learning about relational databases:

  • Field: A single data point, with a particular data type (such as text, integer or date) and an identifying label (such as first_name, product_price or registration_date).
  • Column: A collection of fields of the same data type. In other words, all the fields contain data of the same type and share a common label. For example, a column in a spreadsheet.
  • Row/record: A collection of fields that share a common unique identifier, such as user_id or product_id. For example, a row in a spreadsheet.   The intersection of a row and column corresponds to a field.
  • Table: One or more records containing one or more fields.
  • Relational Database: One or more tables. Where there exist multiple tables in the same database, relationships can be built between records in the different tables.
  • Entity relationships: Connections between items stored across different tables in the same database.

Other important concepts

Two other essential ideas in relational databases are the following:

  • Database normalisation: Database normalisation is the process of organising data into tables in such a way that the results of using the database are always unambiguous and as intended. In simple terms, this means that any one field (such as product_name) should occur only once in a database, regardless of how many tables the database contains.
  • Relational integrity: Integrity constraints are typically used to enforce proper relationships between tables. For example, if a customer can only have one shipping address, then an integrity constraint can be used to ensure that only one shipping address is entered for each customer.

SQL and CRUD

Structured Query Language (SQL, pronounced see-qull) is a programming language that is used to manage relational databases and perform various operations on the data in them.

The term CRUD in databases stands for Create, Read, Update, and Delete. These are the four basic operations you can perform on data stored in a database. Here's a simple explanation of each operation along with its corresponding SQL statement:

Create

This operation adds new records into a database table. The SQL statement INSERT INTO is used for this operation. For example, to add a new user to a table called users.

INSERT INTO users (first_name, last_name, age) VALUES ('John', 'Doe', 30);

Read

This operation retrieves data from a database table. You can read a single record, multiple records, or all the records from a table. The SQL SELECT statement is used for this operation. For example, to read all records from a table called users:

SELECT * FROM users;

Update

This operation modifies existing records in a database table. You can change one or more fields in one or more records. The SQL UPDATE statement is used for this operation. For example, to update the age of a user with a specific user_id in a table called users:

UPDATE users SET age = 31 WHERE user_id = 1;

Delete

This operation removes existing records from a database table. You can delete a single record, multiple records, or all the records from a table. The SQL DELETE statement is used for this operation. For example, to delete a user with a specific user_id from a table called users:

DELETE FROM users WHERE user_id = 1;
