Responsive breakpoints

By default, Tailwind provides a number of responsive breakpoints. These enable you to define specific screen sizes at which styles should be applied differently.

Each has a corresponding minimum width value. For example, the md breakpoint targets screens with a minimum width of 768px.

To apply styles only at specific breakpoints and above, you prefix utility classes with the breakpoint name followed by a colon (:).

For example, if you want a button to have a blue background on medium screens and larger, you'd use the class bg-blue-500 md:bg-green-500. Here:

  • bg-blue-500 applies the blue background by default (on all screens).
  • md:bg-green-500 overrides the background color to green specifically on medium screens (and larger) due to the md prefix.

The container class

Tailwind provides a container class that links an HTML element's width to the current breakpoint. This class sets the max-width of an element to match the min-width of the current breakpoint.

Creating a Grid Layout with TailwindCSS

Follow these steps:

  1. In the /pages, crewte a new component GridExample.jsx with a simple 3-column layout: 
    import React from 'react';

const GridExample = () => {
    return (
        <section className="grid grid-cols-3 gap-4 p-4">
            <div className="bg-blue-500 p-4 text-white">Column 1</div>
            <div className="bg-red-500 p-4 text-white">Column 2</div>
            <div className="bg-green-500 p-4 text-white">Column 3</div>
        </div>
    );
};
        
export default GridExample;
  2. Import this component into app.jsx.
  3. To make the grid response, update it as follows: 
    <section className="grid grid-cols-1 sm:grid-cols-2 md:grid-cols-3 gap-4 p-4">
  4. Add this new component to app.jsx: 
    import React from 'react';
    import './App.css';
    import GridExample from './GridExample';
        
    function App() {
          return (
            <div className="App">
              <header className="App-header">
                <p>
                  TailwindCSS Grid Example in React
                </p>
                <GridExample />
              </header>
            </div>
          );
        }
        
    export default App;

Theming in TailwindCSS

Theming in TailwindCSS allows you to define a consistent look and feel across your React application by customizing colors, fonts, and other design elements.

  1. Open tailwind.config.js and customize the theme by adding your own colors and fonts. Here, we'll add a custom primary color and a secondary color, as well as a custom font family: 
    module.exports = {
    theme: {
      extend: {
        colors: {
          primary: '#4f46e5', // A shade of purple
          secondary: '#fbbf24', // A shade of yellow
        },
        fontFamily: {
          'body': ['"Open Sans"', 'sans-serif'],
          'display': ['"Roboto"', 'sans-serif'],
        },
      },
    },
    plugins: [],
  }
  2. Now that we have our theme defined, let's use these custom styles in our React components. For example: 
    // src/components/Button.jsx
    const Button = ({ children }) => {
      return (
        <button className="bg-primary hover:bg-primary-dark text-white font-bold py-2 px-4 rounded">
          {children}
        </button>
      );
    };
    
export default Button;
  3. You can apply the custom font to your entire application by adding it to the index.css or directly in your component using Tailwind's fontFamily utility: 
    /* src/index.css */
body {
    @apply font-body;
}
    Or, directly in a component: 
    // Using the custom font in a React component
const CustomText = ({ children }) => {
   return <p className="font-body">{children}</p>;
};

With your theme set up, ensure you use the defined colors and fonts consistently across your React application to maintain a unified look and feel. TailwindCSS's utility-first approach makes it easy to apply your theme by using the custom classes you've defined in the configuration.
