Working with the four-column web page

In this Tutorial, you will work with the sample four-column web page, stylesheet and for of the image files you downloaded in a previous Tutorial.

  1. In Visual Studio Code, open the following two files:   flex-four-columns.html
    flex-four-columns.css
  2. Display the flex-four-columns.html web page in your browser. You can see that:
    • All the content is positioned directly against the edges of the browser window.   That is because no padding (inside spacing) has been added to the container-block parent container elements.
    • All the content is 100% wide, left-to-right, across the browser window.   That is because the item-col-4 child columns have not yet been styled.
  3. In VS Code, in the flex-four-columns.css stylesheet, add the following two sets of padding properties and styles for the container-block div.   Each style rule is inside a media query. 
    /* ========== MAIN CONTENT CONTAINERS ========= */
/* Desktops */
@media all and (min-width:768px) { .container-block { padding: 4% 8% } }

/* Mobiles */
@media all and (max-width:767px) { .container-block { padding: 11% 8% } }
  4. Save the flex-four-columns.css file and view the web page in your browser.

You should now see spacing around the four edges of the container-block parent divs.

Setting up the flexbox child columns

Your next task is to set up the styles of the child columns to create your two-column layout on desktop and laptop screens.

  1. In VS Code, in the flex-four-columns.css stylesheet, add the following width rule to your item-col-4 child column divs within the media query for desktop/laptop screens. 
    /* Desktops: flexbox child columns */
@media all and (min-width: 768px) {
    .item-col-4 { width: 22% }
}
    Note that, on mobile devices, your columns will be the full width of the screen.
  2. Save the flex-four-columns.css file and view the web page in your browser.

As you can see, the child columns still fill the full width of the screen on desktops/laptops.

Setting up the flexbox parent element

For the web browser to display your item-col-4 child columns at the set width of 22%, you need to make their parent container, naned container-flexbox, a flexbox element.

  1. In VS Code, in the flex-four-columns.css stylesheet, add the following set of rules to your container-flexbox element. 
    /* Flexbox parent container */
.container-flexbox {
    display: flex;
    justify-content: space-between;
    flex-wrap: wrap; 
}
  2. Save the flex-four-columns.css file and view the web page in your browser.

Your child elements should now display correctly in a four-column layout on desktops/laptop screens.

Use your web browser's screen resizing feature to confirm that your layout displays as a single column on mobile-sized screens.

Adding coloured backgrounds

Next, you will add coloured backgrounds to your two-column layout. First, to the parent container, and then to individual child elements.

Adding a coloured background to the parent container

Follow these steps

  1. In VS Code, in the flex-four-columns.html web page, add the bg-dark class to the first parent container.
  2. Save the web page and view the result in your browser.
  3. For the parent container with the bg-dark background, you need to change the colour of the text.   In your flex-four-columns.css stylesheet, near the bottom, you should see the following comment line. 
    /* Colours for sub-headings and paragraphs */
    Add the following code under it: 
    .container-flexbox.bg-dark .item-col-4 * { color: #fff }
  4. Save the web page and view the result in your browser.

Adding a coloured background to a child element

Follow these steps

  1. In VS Code, in the flex-four-columns.html web page, add the bg-dark class to the third item-col-4 child column inside the second parent container.
  2. Save the web page and view the result in your browser.
  3. You need to change the colour of the text.   In your flex-four-columns.css stylesheet, near the bottom you have the following comment line and a CSS style rule. 
    /* Colours for sub-headings and paragraphs */
.container-flexbox.bg-dark .item-col-4 * { color: #fff }
    Update this by adding a new style rule for child columns. 
    /* Colours for sub-headings and paragraphs */
.container-flexbox.bg-dark .item-col-4 * { color: #fff }
.container-flexbox .item-col-4.bg-dark * { color: #fff }
  4. You need to add some padding to the child column with the coloured background.   In the web page, along with the bg-dark class, add the item-col-padding class to the child element.
  5. Save the web page and stylesheet, and view the result in your browser.

Uploading your files to GitHub

After finishing your web pages and stylesheets (and removed any errors), you are now ready to upload them to your account on GitHub.

  1. Open a new tab in your web browser and go to GitHub.com. If you are not already signed in to your GitHub account, sign in now.
  2. Select or drag-and-drop the following files to upload them to your GitHub account:   flex-four-columns.html
    flex-four-columns.css
    sport-1.jpg
    sport-2.jpg
    sport-3.jpg
    sport-4.jpg
  3. Finally, scroll down to the bottom of the GitHub screen, enter a short message in the Commit changes box and click the Commit changes button.

Your web pages and stylesheets are now published on GitHub at web addresses similar to the following, where username is the username you have chosen for your GitHub account:

https://username.github.io/flex-four-columns.html

It may take a few minutes for your uploaded files to appear on GitHub.
