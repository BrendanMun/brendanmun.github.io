Create the project folders

Your first step is to create folders that will store the HTML files, CSS files, and images needed for these three templates.

  1. If a folder named websites does not already exist on your computer, create it now.
  2. Inside the websites folder, create the following sub-folders if they do not already exist:   C:\websites\assets
    C:\websites\assets\css
    C:\websites\assets\img

Download the three HTML files

Follow these steps to download the three HTML files to your websites folder.

  1. Go to the first HTML file at this web address:  https://ibat-web-dev.github.io/home-page-sample-1.html  
  2. Right-click anywhere on the page (except on an image) and choose View page source (Chrome or Mozilla Firefox).
  3. Right-click on the displayed text and choose Save as… (Chrome) or Save Page As… (Mozilla Firefox).
  4. In the File name: box, change the name of the file to home-page-sample-1.html.
  5. In the Save as type: dropdown list, change from the default value Webpage, Complete to Web Page, HTML Only.
  6. Save the file as home-page-sample-1.html file in your C:\websites folder.
  7. Repeat these steps for the following two other HTML files:  https://ibat-web-dev.github.io/home-page-sample-2.html  https://ibat-web-dev.github.io/home-page-sample-3.html 

You should now have three HTML files saved in your C:\websites folder.

Download the three CSS files

You will now download the three CSS files to the C:\websites\assets\css sub-folder.

  1. In your web browser, open the first CSS file at this web address:  https://ibat-web-dev.github.io/assets/css/home-page-sample-1.css
  2. Right-click on the displayed CSS file and choose Save as… (Chrome) or Save Page As… (Mozilla Firefox).
  3. Save the home-page-sample-1.css file in your C:\websites\assets\css folder.
  4. Repeat these steps for the following two other CSS files:  https://ibat-web-dev.github.io/assets/css/home-page-sample-2.css  https://ibat-web-dev.github.io/assets/css/home-page-sample-3.css 

You should now have three CSS files saved in your C:\websites\assets\css sub-folder.

Download the image files

Follow these steps to download the various image files to the C:\websites\assets\img sub-folder.

  • In each case, right-click on the image and choose Save image as… (Google Chrome) or Save Image As… (Mozilla Firefox). 

Customise your selected sample home page

Review the three versions of the sample home pages in your web browser. On a Windows PC, the three files should be at the following locations:

file:///C:/websites/home-page-sample-1.html
file:///C:/websites/home-page-sample-2.html
file:///C:/websites/home-page-sample-3.html

Which one of the three sample home page versions is closest to the one you would like for your own website?

Follow the steps below to customise it according to your requirements.

  1. In your text editor, open your favourite one of these three HTML files:  home-page-sample-1.html
    home-page-sample-2.html
    home-page-sample-3.html 
  2. For the HTML file you have selected and opened, save it with this new name:  index.html
  3. Open the corresponding stylesheet for the selected HTML in the assets/css sub-folder, and save it with this new name:  home-page.css
  4. In the <head> section of the index.html file, link this file to the stylesheet named home-page.css.
  5. Also in the <head> section, you can remove the 'note' style.
  6. Neat the top of the <body> section, you can remove the text that includes links to alternative versions of the home web page.
  7. You can now work with the index.html and home-page.css files without affecting the sample files you have downloaded.
  8. Make any changes you require to the text and images in the index.html file, and to the styles in the home-page.css stylesheet file. Save both files when finished.

You can ignore the three HTML and three CSS files you downloaded.

Update the meta tags

In the <head> of your index.html web page, within the <title> and <author> meta tags, you can see the name 'Mary Smith.'

Replace this with your own name, and save the HTML file.

Review the Font Awesome 4 icon stylesheet

In the <head> of your privacy.html web page, under the link to the home-page.css stylesheet, is the link to another stylesheet: the CSS file for the Font Awesome icon collection. You need not make any changes to this.

This CSS file is required to create the icons in the footer of the index.html web page.

You can see a full list of Font Awesome icons at the web address below:

  • Font Awesome 4 Icons

You can find some guidance on using Font Awesome icons below:

  • Font Awesome 4 Examples

As with any Google Fonts you may use in your web pages, the Font Awesome icons will display on your computer only when you are connected to the Internet.

Update the Google Analytics ID

Near the top of your HTML file, just before the closing </head> tag, you can see a sample Google Analytics Tracking Code.

Each Google Tracking Code has a unique ID in its first and last line. Replace the sample Google Tracking ID with your own Google Tracking ID. The instructions for viewing your website’s unique Google Tracking ID are here.

When finished, save the index.html file.

Update your contact details in the footer

At the bottom of the index.html file is the footer section for the web page. It appears as three columns on desktop screens and as a single column on mobile phones. The footer section contains three parts:

  • Sitemap: This contains links to three other pages on your website: the Home, Portfolio and Privacy pages. In this example, the Portfolio page is the current web page.  You need not make any changes to this part of the footer. 
  • About Me: This section is for showing a brief description of your website.  Change the name from 'Mary Smith' to your own name, and make any other changes to the text as you require. 
  • Contact: This lists your email address and other the various ways you would like people to get in touch with you. Amend these contact details to your own. You can add or remove different social media contact points, such as Twitter, Facebook, YouTube or Instagram, as required. 

When finished updating the footer, save the index.html file and view the result in your web browser.

Adding the privacy pop-up code and message

At the bottom of the HTML file, just before the closing </body> tag, copy-and-paste the following ‘pop-up‘ privacy code and message. You can find the instructions for generating this code here.


<script src="https://cdn.jsdelivr.net/npm/cookieconsent@3/build/cookieconsent.min.js" data-cfasync="false"></script>
<script>
window.cookieconsent.initialise({
  "palette": {
    "popup": {
      "background": "#000"
    },
    "button": {
      "background": "#f1d600"
    }
  },
  "content": {
    "href": "https://username.github.io/privacy.html"
  }
});
</script>
</body>
</html>

Near the end of this code is the sample web address of https://username.github.io/privacy.html.

Change this to the web address of the privacy page on your own website.

Validate your HTML file

To check your index.html file is correct, use the official W3C Markup Validation Service. Follow these steps.

  1. Go to this web page: https://validator.w3.org.
  2. Click the Validate by Direct Input tab.
  3. Copy and paste your index.html file into the box named Enter the Markup to validate.
  4. Click the Check button.
  5. If you see any errors, return to your index.html, fix the errors, save the file, and copy the entire file again.  In the HTML Validator, click the Back button of your web browser to again display the Validate by Direct Input tab. Click once in the tab and paste in your corrected HTML file. Your new, pasted-in file will replace the earlier version. Finally, click the Check button.

Validate your CSS file

To check your home-page.css file is correct, use the official W3C CSS Validation Service. Follow these steps.

  1. Go to this web page: https://jigsaw.w3.org/css-validator.
  2. Click the By direct input tab.
  3. Copy and paste your home-page.css file into the box named Enter the CSS you would like validated.
  4. Click the Check button.
  5. If you see any errors, return to your home-page.css file, fix the errors, save the file, and copy the entire file again.  In the CSS Validator, click the Back button of your web browser to again display the By direct input tab. Click once in the tab and paste in your corrected CSS file. Your new, pasted-in file will replace the earlier version. Finally, click the Check button.

Note: The CSS Validator cannot process fluid typography styles. So ignore any messages such as those below.

Uploading your home page files to GitHub

The final step is to upload your project to GitHub. Follow these steps.

After validating your web page and stylesheet, you are now ready to upload them to your account on GitHub.

  1. Sign in to your account at GitHub.com. At the left of the screen, you can see a list of your repositories.
  2. On your GitHub home page, click the name of the repository that holds your web pages. Its name will look as follows, where username is your chosen username on GitHub.   username.github.io  
  3. On the next screen displayed, near the centre of the screen, click the Upload button.
  4. Select or drag-and-drop the index.html file and the assets folder to upload them to your GitHub account.
  5. After uploading the file and folder, scroll down to the bottom of the screen, enter a short message in the Commit changes box and click the Commit changes button.

Your web page is now published on GitHub at a web address similar to the following, where username is the username you have chosen for your GitHub account:

https://username.github.io/index.html

or, simply:

https://username.github.io

It may take a few minutes for your uploaded files to appear on GitHub.
